April 2018 - present

I have always been fond of the idea of social networks since they started thriving in the mid 2000s, but the privacy violations have been a serious matter of concern for me and a lot of people. The non-existence of a really private social network was the motivation for me to start building STiiiCK. This mobile application is the biggest project that I have been working on for over 3 years. I got the idea of STiiiCK in summer of 2017, then I started immediately researching & learning all the technologies & skills that I would need to make this idea a reality. STiiiCK is using a custom end-to-end encryption protocol tailored for social networks - the Stick Protocol. STiiiCK is now available for iOS and Android.

Stick Protocol

An End-to-End Encryption Protocol Tailored for Social Network Platforms

End-to-end encryption has become a de facto standard in messengers, especially after the outbreak of the highly secure messaging protocol ‚Äì Signal. However, this high adoption of secure end-to-end communications has been limited to messengers, and has not yet seen a noticeable trace in social network platforms, despite the increase in users' privacy violations. I developed the Stick protocol - an end-to-end encryption protocol specifically designed for social networks. Stick is the encryption protocol powering STiiiCK. The Stick protocol supports re-establishable many-to-many encryption sessions in an asynchronous and multi-device setting while preserving forward secrecy and introducing backward secrecy. The Stick protocol is open-source available on Github. Evaluation üìà results of the Stick protocol shows that it causes no noticeable compromise on usability or performance. Steganography

 February 2019

A program to encode and decode a text message or an image in another image

February 2019

This is a simple steganography system built with C. This system has a command-line interface that allows its users to hide/encode a message or an image into another image, then reveal/decode the text or image from the encoded image. It also allows the user to choose the Most-Significant-Bits count of the RGB values of an image to encode. Encoding more bits will alter the output encoded image more, but also will produce a better quality decoded image. Example input and output can be seen from here.
